# 项目需求分析

> CampusTrade 校园二手交易平台 | 2026-05-22

---

## 1. 项目背景与意义

### 1.1 背景

高校校园内存在大量的闲置物品流通需求：毕业生离校前的物品清仓、新生入学时的用品采购、学生之间教材和电子产品的流转等。目前学生主要依赖 QQ 群、微信群、校园墙等方式进行交易，存在以下问题：

- **信息分散：** 交易信息散布在多个群聊中，难以检索和筛选
- **信任缺失：** 群内成员身份不明确，存在校外人员混入风险
- **无持久性：** 聊天记录滚动覆盖，发布的商品信息很快被淹没
- **无管理机制：** 缺乏内容审核，违规商品难以管控

### 1.2 意义

本平台通过 Web 技术构建一个专属于本校学生的二手交易系统，实现：

1. **统一入口：** 所有校内二手交易信息集中展示，支持搜索筛选
2. **身份可信：** 仅限本校学生注册使用，降低交易风险
3. **审核机制：** 管理员对发布商品进行审核，保障平台内容质量
4. **信息持久：** 商品信息结构化存储，不会因时间推移而丢失

## 2. 可行性分析

### 2.1 技术可行性

| 维度 | 分析 |
|------|------|
| 前端技术 | Vue 3 + Element Plus 生态成熟，组件库完善，开发效率高 |
| 后端技术 | Spring Boot + MyBatis-Plus 是 Java Web 开发主流方案，文档和社区资源丰富 |
| 数据库 | MySQL 8.0 满足中小规模数据存储需求，Redis 提供缓存和会话管理能力 |
| 实时通信 | Spring WebSocket 原生支持，无需引入第三方服务 |
| 部署 | 可部署于单台云服务器（2C4G），满足校园规模并发需求 |

**结论：** 技术方案成熟可行，个人开发者可独立完成。

### 2.2 经济可行性

- 开发工具：开源免费（VS Code / IntelliJ IDEA Community、MySQL Community、Redis Community）
- 服务器：阿里云学生机或本地虚拟机，成本极低
- 第三方服务：MVP 阶段不接入支付、短信等付费服务

**结论：** 开发和运行成本可控，经济可行。

### 2.3 操作可行性

- 目标用户（高校学生）具备基本的 Web 应用使用能力
- UI 采用主流电商布局模式（分类导航 + 搜索 + 商品卡片），学习成本低
- 管理端面向少数管理员，操作简单明确

**结论：** 用户操作门槛低，操作可行。

## 3. 用户角色分析

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| 普通用户（买家） | 浏览和购买二手商品的学生 | 搜索商品、查看详情、联系卖家、收藏、下单 |
| 普通用户（卖家） | 发布和出售二手商品的学生 | 发布商品、管理发布、查看审核状态、确认成交 |
| 管理员 | 平台内容管理者 | 审核商品、管理用户、管理分类、查看数据 |

### 3.1 普通用户用例

- **注册账号：** 填写用户名、密码、昵称、手机号完成注册
- **登录系统：** 使用用户名和密码登录，获取 JWT 令牌
- **浏览商品：** 首页查看最新/热门商品，分类浏览，关键词搜索，按条件筛选
- **查看详情：** 查看商品完整信息、图片、卖家信息
- **收藏商品：** 收藏感兴趣的商品，在收藏列表中统一查看
- **发布商品：** 填写商品信息、上传图片、选择分类后提交审核
- **管理发布：** 查看自己发布的商品及其审核状态和销售状态
- **私信沟通：** 与买家/卖家进行实时聊天
- **交易下单：** 对在售商品下单，确认成交或取消订单
- **管理个人信息：** 修改昵称、头像、联系方式

### 3.2 管理员用例

- **登录管理端：** 使用管理员账号登录后台管理系统
- **商品审核：** 查看待审核商品列表，预览商品详情，通过或驳回（含驳回原因）
- **用户管理：** 查看所有注册用户，对违规用户进行禁用/启用操作
- **分类管理：** 新增、编辑、删除商品分类
- **数据概览：** 查看平台核心数据统计（用户数、商品数、交易数）

## 4. 功能性需求

### 4.1 用户认证模块

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-AUTH-01 | 用户注册：用户名（唯一）、密码（≥6位）、昵称、手机号 | P0 |
| F-AUTH-02 | 用户登录：用户名+密码，返回 JWT token | P0 |
| F-AUTH-03 | 获取当前用户信息：根据 token 返回用户基本信息 | P0 |
| F-AUTH-04 | 管理员登录：仅 role=admin 的用户可登录管理端 | P0 |

### 4.2 商品模块

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-PROD-01 | 发布商品：标题、描述、售价、原价、成色、分类、图片（1-9张） | P0 |
| F-PROD-02 | 编辑商品：仅发布者可修改 | P0 |
| F-PROD-03 | 删除商品：软删除（状态置为 hidden） | P0 |
| F-PROD-04 | 商品列表：分页展示，支持分类/成色/价格筛选、关键词搜索、排序 | P0 |
| F-PROD-05 | 商品详情：完整信息+图片轮播+卖家信息+浏览量统计 | P0 |
| F-PROD-06 | 仅展示审核通过的活跃商品 | P0 |
| F-PROD-07 | 图片上传：支持 jpg/png/webp，≤5MB，前端压缩 | P0 |

### 4.3 商品审核模块

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-AUDIT-01 | 待审核列表：管理员查看所有审核状态为 pending 的商品 | P0 |
| F-AUDIT-02 | 审核通过：商品状态变为 approved，用户端可见 | P0 |
| F-AUDIT-03 | 审核驳回：填写驳回原因，商品状态变为 rejected，用户可在"我的发布"中查看原因 | P0 |
| F-AUDIT-04 | 新发布商品自动进入待审核状态 | P0 |

### 4.4 分类模块

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-CAT-01 | 用户端获取全部分类（树形结构） | P0 |
| F-CAT-02 | 管理员新增/编辑/删除分类 | P0 |
| F-CAT-03 | 支持一级和二级分类 | P0 |

### 4.5 收藏模块

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-FAV-01 | 收藏商品（同一用户对同一商品不可重复收藏） | P0 |
| F-FAV-02 | 取消收藏 | P0 |
| F-FAV-03 | 收藏列表（分页，支持取消收藏） | P0 |
| F-FAV-04 | 收藏数统计（product.fav_count） | P0 |

### 4.6 私信模块

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-CHAT-01 | 创建会话：买家对某商品发起与卖家的私信 | P0 |
| F-CHAT-02 | 发送消息：文本消息（WebSocket 实时推送） | P0 |
| F-CHAT-03 | 历史消息：分页查看历史聊天记录 | P0 |
| F-CHAT-04 | 会话列表：按最后消息时间排序，显示未读数 | P0 |
| F-CHAT-05 | 同一买家-卖家-商品组合不重复创建会话 | P0 |

### 4.7 订单模块

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-ORD-01 | 下单：买家对在售商品下单，商品变为 reserved | P0 |
| F-ORD-02 | 确认成交：卖家确认，商品变为 sold | P0 |
| F-ORD-03 | 取消订单：买卖双方均可取消，商品恢复 active | P0 |
| F-ORD-04 | 订单列表：我买到的 / 我卖出的 | P0 |
| F-ORD-05 | 不可重复下单同一商品 | P0 |
| F-ORD-06 | 不可对自己发布的商品下单 | P0 |

### 4.8 用户管理模块（管理端）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-UM-01 | 用户列表（分页） | P0 |
| F-UM-02 | 启用/禁用用户 | P0 |
| F-UM-03 | 禁用用户无法登录 | P0 |

### 4.9 数据概览模块（管理端）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-DASH-01 | 统计卡片：用户总数、商品总数、今日新增商品、订单总数 | P0 |

| F-DASH-03 | 最近发布商品列表 | P1 |

## 5. 非功能性需求

| 类别 | 要求 |
|------|------|
| **性能** | 列表页首屏加载 ≤ 1s，API 响应 ≤ 500ms（P95） |
| **可用性** | 系统可用性 ≥ 99%（计划内维护除外） |
| **安全性** | 密码 BCrypt 加密、JWT 令牌过期机制、SQL 注入防护、XSS 防护 |
| **可维护性** | 严格分层架构（Controller-Service-Mapper），代码注释规范 |
| **兼容性** | 支持 Chrome / Safari / Edge 最近 2 个主版本 |
| **可扩展性** | 预留租赁和技能服务模块扩展接口 |
