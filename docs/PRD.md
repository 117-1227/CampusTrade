# PRD — CampusTrade 产品需求文档

> 版本：v1.0 MVP | 最后更新：2026-05-22
> 论文题目：基于 Spring Boot + Vue 的校园二手交易平台设计与实现

## 1. 产品定位

面向高校学生群体的校园二手交易平台。同校学生之间交易，距离近、信任成本低、无需快递物流。

**核心价值：** 提高校园闲置物品流通效率，降低学生交易成本。

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 普通用户（学生） | 浏览商品、发布商品、收藏、私信、交易 |
| 管理员 | 商品审核、用户管理、分类管理、数据概览 |

## 3. 功能清单

### 3.1 用户端功能

#### P0 — 核心功能

| 模块 | 功能 | 描述 |
|------|------|------|
| 用户认证 | 注册 | 用户名 + 密码 + 昵称 + 手机号（或邮箱） |
| 用户认证 | 登录 | JWT token 认证 |
| 商品发布 | 发布商品 | 标题、描述、售价、原价、成色、分类、图片（最多9张） |
| 商品发布 | 图片上传 | 前端压缩 + 后端存储，支持多图 |
| 商品发布 | 编辑/删除 | 仅本人可操作 |
| 商品浏览 | 首页 | 分类导航 + 最新商品 + 热门商品 |
| 商品浏览 | 商品列表 | 分页、分类筛选、成色筛选、价格排序、关键词搜索 |
| 商品浏览 | 商品详情 | 图片轮播、商品信息、卖家信息、收藏入口、聊天入口 |
| 收藏 | 收藏/取消 | 详情页和列表页快捷操作 |
| 收藏 | 收藏列表 | 分页展示、支持取消收藏 |
| 私信 | 会话管理 | 买家→卖家发起对话 |
| 私信 | 实时聊天 | WebSocket 实时消息推送 |
| 交易 | 下单 | 买家下单，商品变为已预定 |
| 交易 | 确认成交 | 卖家确认，商品变为已售出 |
| 交易 | 取消订单 | 买卖双方均可取消，商品恢复在售 |
| 个人中心 | 我的发布 | 分状态查看（在售/审核中/已驳回/已售） |
| 个人中心 | 我的收藏 | 收藏商品列表 |
| 个人中心 | 我的订单 | 我买到的 / 我卖出的 |

#### P1 — 体验增强（MVP 后期）

| 功能 | 描述 |
|------|------|
| 商品浏览量统计 | 记录商品查看次数 |
| 搜索历史 | localStorage 存储最近搜索 |
| 个人信息编辑 | 修改头像、昵称、联系方式 |

### 3.2 管理端功能

| 模块 | 功能 | 描述 |
|------|------|------|
| 管理员认证 | 登录 | 管理员账号由系统初始化创建 |
| 商品审核 | 审核列表 | 待审核商品列表，支持分页 |
| 商品审核 | 通过/驳回 | 通过→商品上架；驳回→填写驳回原因，通知用户 |
| 用户管理 | 用户列表 | 查看所有注册用户 |
| 用户管理 | 启用/禁用 | 禁用后用户无法登录 |
| 分类管理 | 新增分类 | 支持一级和二级分类 |
| 分类管理 | 编辑/删除 | 有商品的分类不可删除 |
| 数据概览 | 首页统计 | 用户总数、商品总数、今日新增、交易总数 |

### 3.3 明确不做（v1）

- 在线支付（微信/支付宝）
- 用户评价/信用体系
- 闲置租赁
- 技能服务交易
- 微信小程序
- 推送通知
- 举报/投诉

## 4. 核心用户流程

```
用户端：
注册/登录 → 浏览首页 → [搜索/筛选商品] → 查看详情
                                              ├── 收藏
                                              ├── 联系卖家 → 私信 → 下单 → 卖家确认成交
                                              └── 分享

卖家端：
登录 → 发布商品 → 等待审核 → 审核通过上架 → 收到私信 → 协商 → 确认成交

管理端：
管理员登录 → 商品审核（通过/驳回） / 用户管理 / 分类管理 / 数据概览
```

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 响应时间 | 列表页首屏 < 1s，详情页 < 500ms |
| 并发 | 支持 500 同时在线 |
| 安全 | 密码 BCrypt 加密、JWT 令牌、SQL 注入防护、XSS 防护、CSRF 防护 |
| 文件 | 图片 ≤ 5MB，前端压缩，后端校验类型和大小 |
| 兼容 | Chrome/Safari/Edge，桌面端为主 |
| 代码质量 | Spring Boot 三层架构、Vue 3 Composition API、全栈无 any |
